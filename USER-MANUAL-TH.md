## 1. เริ่มต้นใช้งาน

### การเข้าสู่ระบบ
1. เปิด `https://spa.siamepos.co.uk` (หรือ URL ของสปาคุณ) บนแท็บเล็ต แล็ปท็อป หรือมือถือ
2. กดรหัส PIN ของคุณ ค่าเริ่มต้นของผู้ดูแลคือ **1234** — ควรเปลี่ยนทันที (ดูหัวข้อ [จัดการพนักงาน](#8-จัดการพนักงาน))
3. คุณจะเข้าสู่ระบบสปา

### รายการที่ต้องตั้งค่าก่อนเริ่มใช้งานวันแรก
ก่อนรับลูกค้าครั้งแรก ให้ทำรายการเหล่านี้ในเมนู **Admin (ผู้ดูแล)**:

- [ ] **🔧 Settings (ตั้งค่า)** — กรอกชื่อสปา ที่อยู่ เบอร์โทร อีเมล — ข้อมูลเหล่านี้จะปรากฏบนใบเสร็จ อีเมล และ Z-Report
- [ ] **👥 Therapists (พนักงานสปา)** — เพิ่มพนักงานแต่ละคน พร้อมรหัส PIN ที่ไม่ซ้ำกัน, บทบาท (admin / manager / reception / therapist), รูปถ่าย และความเชี่ยวชาญ (specialisms)
- [ ] **📅 Rota (ตารางทำงาน)** — กำหนดตารางทำงานประจำสัปดาห์ของแต่ละคน
- [ ] **🛁 Rooms (ห้อง)** — เพิ่มห้องนวดของสปา (เช่น "Room A", "Couples Suite")
- [ ] **💆 Treatments (บริการ)** — เพิ่ม Treatment แต่ละรายการ พร้อมระยะเวลา ราคา และหมวดหมู่ ตั้งค่า VAT ถ้ามี
- [ ] **⚙️ Booking (การจอง)** — ตั้งเวลาเปิด-ปิดร้าน, ช่วงเวลานัด (slot), และ tip suggestions (แนะนำทิป)
- [ ] **🌐 Online Booking** — ตั้งค่านโยบายมัดจำ ถ้าจะรับจองออนไลน์

หลังจากนี้ ระบบพร้อมรับลูกค้าได้

## 2. งานหน้าเคาน์เตอร์ประจำวัน

หน้าจอหลักคือ **Appointment timeline (ตารางนัดหมาย)** — แสดงเป็นคอลัมน์แนวตั้ง คอลัมน์ละหนึ่งพนักงาน เรียงเวลาจากบนลงล่าง

### สัญลักษณ์บน timeline

| สิ่งที่เห็น | ความหมาย |
|---|---|
| Header คอลัมน์มี **★ 10:00–20:00** สีทอง | พนักงานคนนั้นมีเวลาทำงานพิเศษ (custom-hour override) สำหรับวันนั้น |
| Header คอลัมน์แสดง **11:00–21:00** สีขาว | ตารางปกติของสัปดาห์ (weekly rota) |
| คำว่า **"Off"** | พนักงานไม่ทำงานวันนั้น |
| แถบสีเทาลายเฉียงด้านบน/ด้านล่างคอลัมน์ | เวลานอกช่วงทำงาน — จองไม่ได้ |
| บล็อกสีบนคอลัมน์ | การจอง (สี = สถานะหรือวิธีชำระเงิน) |

### สีของการจอง

| สี | ความหมาย |
|---|---|
| 🔵 ฟ้า | **Booked** — ยืนยันการจองแล้ว ยังไม่เริ่มบริการ |
| 🟢 เขียว | **In progress** — ลูกค้ามาแล้ว กำลังให้บริการ |
| ⬜ เทา | **Completed** — บริการเสร็จแล้ว รอชำระเงิน |
| 🟠 ส้ม | จบบริการ + จ่ายด้วย **Cash (เงินสด)** |
| 🩷 ชมพู | จบบริการ + จ่ายด้วย **Card (บัตร)** |
| 🟢 เขียว | จบบริการ + จ่ายด้วย **Voucher (วอเชอร์)** |
| 🟡 เหลือง | จบบริการ + จ่ายด้วย **Treatwell** |
| 🟣 ม่วง | จบบริการ + จ่ายแบบ **Split (แยกหลายวิธี)** |

### การจองใหม่

**วิธีที่ 1 — แตะช่องว่างบน timeline**
1. แตะเวลาว่างบนคอลัมน์ของพนักงานคนที่ต้องการ
2. หน้าต่างจองจะเปิดขึ้น พร้อมวันที่ เวลา และพนักงานที่กรอกไว้แล้ว
3. เลือก Treatment
4. ค้นหาลูกค้า:
   - พิมพ์ในช่องค้นหา → ลูกค้าเดิมจะปรากฏ → แตะเลือก
   - หรือกด **"+ New client / walk-in (full details)"** เพื่อเพิ่มลูกค้าใหม่ กรอกชื่อ (จำเป็น) + เบอร์โทร อีเมล วันเกิด ผู้ติดต่อฉุกเฉิน GP ความยินยอม GDPR การรับอีเมลโฆษณา และโน้ต
5. แตะ **Book appointment (จองนัด)**

**วิธีที่ 2 — ใช้ปุ่ม +New**
แตะปุ่ม **+ New** บนแถบเครื่องมือ เลือก Treatment ก่อน แล้วเลือกวันเวลา จากนั้นเลือกลูกค้า แล้วกดบันทึก

### แก้ไขหรือเลื่อนนัด
ดับเบิลคลิกที่การจองบน timeline เพื่อเปิดหน้าต่างแก้ไข เปลี่ยน Treatment เวลา พนักงาน สถานะ หรือโน้ต กดบันทึก

### เปลี่ยนสถานะ (โดยไม่ต้องชำระเงิน)
ในหน้าต่างแก้ไข เปลี่ยน dropdown **Status**:
- **Booked** → **In progress** เมื่อลูกค้ามาถึงและพาเข้าห้อง
- **In progress** → **Completed** เมื่อบริการจบ (จากนั้นไปที่ Checkout)
- **Cancelled** ถ้าลูกค้ายกเลิกก่อนมา
- **No-show** ถ้าลูกค้าไม่มา

### การยกเลิกและคืนเงิน
- **ลูกค้า walk-in**: เปลี่ยนสถานะเป็น "Cancelled" — จบ
- **จองออนไลน์ที่จ่ายมัดจำแล้ว**: ลูกค้ายกเลิกเองได้ผ่านลิงก์ในอีเมล (คืนเงินถ้ายังในช่วงที่อนุญาต, ยึดมัดจำถ้านอกช่วง) หรือคุณคืนเงินด้วยตนเองได้จาก **Admin → 🌐 Online Booking**

## 3. การชำระเงิน (Checkout)

เมื่อสถานะของลูกค้าเป็น **In progress** หรือ **Completed** แตะการจองบน timeline → กด **Checkout** (หรือ **Bill**)

### หน้าจอ Checkout

```
┌──────────────────────────────────────┐
│ Treatment               £55.00        │
│ Tip                      £0.00        │
│ Total                   £55.00        │
│                                       │
│ 💳 Deposit paid online   −£25.00     │  ← แสดงเฉพาะเมื่อจองออนไลน์ + จ่ายมัดจำ
│ ┌──────────────────────────────────┐ │
│ │ Balance due now    £30.00         │ │  ← จำนวนที่ลูกค้าต้องจ่ายตอนนี้
│ └──────────────────────────────────┘ │
│                                       │
│ [Tip: 10% 12.5% 15% No tip Custom]   │
│                                       │
│ Payment method (วิธีชำระ):           │
│ [Cash] [Card] [🎁 Voucher] [🌐 Treatwell] [⇄ Split] │
└──────────────────────────────────────┘
```

### ชำระด้วยวิธีเดียว
- **Cash (เงินสด)**: แตะ Cash → บิลปิด ถ้ามีมัดจำ ระบบจะบันทึกเป็น "deposit £25 + cash £30" อัตโนมัติ
- **Card (บัตร)**: แตะ Card → บิลปิด ถ้ามีเครื่องรูดบัตร ให้รูดต่างหาก
- **🎁 Voucher**: แตะ Voucher → ใส่รหัสวอเชอร์ → Check → กดยืนยัน Redeem
- **🌐 Treatwell**: แตะ Treatwell (ลูกค้าจ่ายผ่าน Treatwell แล้ว — แค่ปิดบิลในระบบของเรา)

### Split — แยกหลายวิธี
แตะ **⇄ Split** หน้าต่าง Split modal จะเปิดขึ้น พร้อม header แสดงยอดรวมและยอดคงเหลือ

1. มี 2 แถวเริ่มต้น (Cash และ Card)
2. แต่ละแถว: เลือกวิธี (Cash / Card / 🎁 Voucher) แล้วใส่จำนวนเงิน
3. แถว **🎁 Voucher** จะขยายให้กรอกรหัสวอเชอร์ — ใส่รหัส แล้วแตะ Check
4. Header แสดง **"Remaining £X"** สีขาว หรือ **"Over by £X"** สีแดง จนกว่ายอดรวมจะตรงกับยอดบิลเป๊ะๆ
5. เมื่อยอดตรง ปุ่ม **"Take £X & close bill"** จะเปลี่ยนเป็นสีทอง
6. แตะ — บิลปิดพร้อมบันทึกรายละเอียดการชำระแยกตามวิธี (รายงานจะแสดงแต่ละส่วนตามวิธีที่จ่ายจริง — ไม่มีเงินหายไปอยู่ในหมวด "split" ที่คลุมเครือ)

### Tip (ทิป)
ก่อนเลือกวิธีชำระ ให้กำหนดทิป:
- แตะปุ่มเปอร์เซ็นต์ (10% / 12.5% / 15%) เพื่อใส่ทิปแบบแนะนำตามราคา Treatment
- หรือแตะ **No tip** หรือใส่จำนวน **Custom** เอง
- ทิปจะบันทึกแยกจากยอดบริการในรายงาน

### แก้ไขวิธีชำระเงิน (หลังปิดบิลแล้ว)
ถ้ากดผิด เช่น ลูกค้าจ่ายบัตรแต่กด Cash ไป — เปิดหน้าต่างแก้ไขการจอง (Edit modal) สำหรับการจองที่สถานะ **Completed** จะเห็นแถบสีเทา **"💰 Paid by Cash · £70 [Edit]"** กดปุ่ม **Edit** → เลือกวิธีที่ถูก → กดบันทึก — ผู้ดูแลและผู้จัดการเท่านั้นที่แก้ได้

## 4. การจองออนไลน์ผ่านเว็บไซต์

ระบบสปามี widget สำหรับจองออนไลน์ที่ลูกค้าใช้จากเว็บไซต์ของคุณได้

### ฝัง widget ในเว็บไซต์
ไปที่ **Admin → 🔗 Embed Codes** เพื่อดูโค้ดสำหรับฝัง:

**ตัวเลือก 1 — ปุ่มสำเร็จรูป (แนะนำ):**
```html
<script src="https://spa-api.siamepos.co.uk/booking-widget.js" defer></script>
<div id="siamespa-booking"></div>
```
วางโค้ดนี้ในตำแหน่งที่อยากให้ปุ่ม "Book your treatment" ปรากฏ

**ตัวเลือก 2 — ปุ่มของคุณเอง:**
```html
<script src="https://spa-api.siamepos.co.uk/widget.js" defer></script>
<button onclick="SiamEPOSSpa.open()">Book now</button>
```

### ลูกค้าเห็นอะไรบ้าง
1. **Treatment** — เลือกจากรายการสด
2. **Therapist** — เห็นรูปและความเชี่ยวชาญ (หรือ "Any available")
3. **Date + time** — ตัวเลือกเวลาว่าง กรองตามตารางพนักงานที่เลือก
4. **Details** — ชื่อ เบอร์โทร อีเมล ความยินยอม GDPR
5. **Payment** — ใส่บัตร Stripe (ถ้าตั้งนโยบายมัดจำไว้)
6. **Confirmation** — เลขที่จอง + ลิงก์จัดการ

### นโยบายมัดจำ (Deposit policy)
ไปที่ **Admin → 🌐 Online Booking** เพื่อตั้งค่า:

- **Deposit model**:
  - **No deposit** — ลูกค้าจองออนไลน์โดยไม่ต้องจ่าย
  - **Fixed amount** — ค่ามัดจำคงที่ทุกการจอง (เช่น £25) — ใช้บ่อยที่สุด
  - **Percentage** — เปอร์เซ็นต์ของราคา Treatment (เช่น 25%) — Treatment แพงก็มัดจำมากขึ้น
  - **Full prepay** — จ่ายเต็มจำนวนออนไลน์ ไม่ต้องจ่ายอะไรที่ร้านอีก

- **Cancel window (hours)** — ก่อนวันนัดกี่ชั่วโมงที่ลูกค้ายังยกเลิกได้ฟรี
- **Cancellation policy text** — ข้อความนโยบายที่แสดงให้ลูกค้าเห็นบนหน้าจัดการ

### ดูการจองออนไลน์แบบเรียลไทม์
แท็บ **🌐 Online Booking** ยังแสดงรายการจองออนไลน์ทั้งหมดแบบเรียลไทม์:
- วันที่ ลูกค้า Treatment พนักงาน
- ป้ายสถานะการชำระเงิน (Deposit paid / Refunded / Forfeit / Fully paid)
- ลิงก์ไปยัง Stripe dashboard
- ปุ่มคืนเงิน (Refund)

### กระแสเงินสดของมัดจำ
- เงินมัดจำเข้า **บัญชี Stripe ของคุณ** ทันทีที่ลูกค้าจ่ายออนไลน์
- ปรากฏในรายงาน Trading ภายใต้ **🌐 Online deposits** — แยกจากเงินสดในร้าน
- เมื่อลูกค้ามาและจ่ายส่วนที่เหลือที่ร้าน ระบบจะเครดิตมัดจำให้อัตโนมัติ (ลูกค้าจ่ายเฉพาะส่วนที่เหลือ)
- ตอนปิดร้าน Z-Report จะแสดงทั้งสองอย่าง: เงินสดในร้าน + มัดจำผ่าน Stripe

## 5. ระบบจัดการการจองด้วยตนเอง (Customer Portal)

ทุกการจองออนไลน์ที่ยืนยันแล้วจะได้รับลิงก์ "Manage your booking" ในอีเมลยืนยัน หน้านี้ใช้งานได้โดยไม่ต้องล็อกอิน (ใช้ token ที่เซ็นในลิงก์)

### ลูกค้าทำอะไรได้
- **ดูการจอง** — วันที่ เวลา Treatment พนักงาน มัดจำที่จ่าย ยอดที่เหลือ
- **Reschedule (เลื่อนนัด)** — เลือกเวลาใหม่จากตารางว่างของสปา ระบบตรวจสอบกับตารางพนักงานอีกครั้งฝั่ง server
- **Cancel (ยกเลิก)** — คืนเงินถ้าอยู่ในช่วงนโยบาย, ยึดมัดจำถ้านอกช่วง ส่งอีเมลแจ้งทั้งสองกรณี

### ลูกค้าทำอะไรไม่ได้
- เปลี่ยน Treatment (จะทำให้เรื่องราคาซับซ้อน)
- เลื่อนไปเวลาที่พนักงานไม่ได้ทำงาน
- แก้ไขการจองของคนอื่น (token เซ็นและผูกกับการจองนั้นๆ)

## 6. จองทางโทรศัพท์ + ลิงก์ชำระเงิน

บางครั้งลูกค้าโทรมาและอยากจองพนักงาน + Treatment + เวลาที่เจาะจง และอยากจ่ายเลย โดยไม่ต้องเข้าเว็บไซต์

**ขั้นตอน:**
1. พนักงานต้อนรับสร้างการจองในระบบแอดมิน (เหมือนการจองปกติ)
2. ในหน้าต่างแก้ไขการจอง จะเห็น **"💳 Deposit not collected"** พร้อมปุ่มทอง **Request deposit by link**
3. แตะปุ่ม → ระบบสร้างลิงก์ชำระเงิน Stripe
4. แผงจะขยาย แสดง 3 ปุ่ม:
   - **📋 Copy** — คัดลอกลิงก์ไปวางที่อื่น
   - **📱 WhatsApp** — เปิด WhatsApp พร้อมข้อความและลิงก์
   - **✉️ Email** — เปิดแอปอีเมลพร้อมข้อความและลิงก์ส่งถึงลูกค้า
5. ส่งลิงก์ทางช่องที่สะดวก
6. การจองอยู่ในสถานะ **deposit_pending** — กั้นเวลาไว้แต่ยังไม่ยืนยัน
7. ลูกค้ากดลิงก์ → เปิดหน้าจ่ายเงิน → ใส่บัตร → จ่าย
8. การจองเปลี่ยนเป็น **deposit_paid** อัตโนมัติ พร้อมส่งอีเมลยืนยัน

ถ้าลูกค้าทำลิงก์หาย แค่กดปุ่ม **🔄 New link** เพื่อสร้างใหม่

## 7. การเชื่อมต่อกับ Treatwell

ถ้าสปาคุณอยู่บน Treatwell ระบบจะรับการจองอัตโนมัติผ่าน webhook

### การตั้งค่า (ครั้งเดียว)

2. ใน Treatwell partner portal เพิ่ม webhook URL:
   `https://spa-api.siamepos.co.uk/api/treatwell/webhook`
3. ตั้ง header `X-Treatwell-Secret` เป็นค่าเดียวกับ secret

### สิ่งที่เกิดขึ้น
- การจอง Treatwell เข้ามา → ระบบสร้างนัดและลูกค้า ติดป้าย `source='treatwell'`
- ระบบจับคู่ Treatment ด้วยชื่อ (ไม่สนตัวพิมพ์ใหญ่/เล็ก) ถ้าจับคู่ไม่ได้ จะติดป้าย `[unmatched treatment]` ในโน้ตให้ตรวจสอบ
- การยกเลิกผ่าน Treatwell จะเปลี่ยนสถานะนัดเป็น "Cancelled"
- webhook ที่ซ้ำจะถูกข้าม

### ปิดบิล Treatwell
ตอน Checkout จะเห็นปุ่ม **🌐 Treatwell** (แสดงตลอด แต่ใช้จริงเมื่อลูกค้ามาจาก Treatwell) แตะ — บิลปิดโดยไม่รับเงินสด Treatwell จะจ่ายเข้าบัญชีคุณภายหลัง (หักค่าคอมมิชชั่น)

### Treatwell ในรายงาน
- **Admin → 👤 Clients** มีป้าย 🌐 Treatwell เพื่อระบุลูกค้า Treatwell และส่งโปรโมชั่นจองตรง (ค่าคอมต่ำกว่า)
- **Admin → 📈 Reports** มีการ์ด "Booking source" แสดงรายได้ Treatwell เทียบกับการจองตรง

### แบบฟอร์มข้อมูลทางการแพทย์ (กฎหมาย UK บังคับ)

## 11. วอเชอร์ของขวัญ

สปามักขายวอเชอร์เป็นของขวัญ (เช่น คริสต์มาส) หรือเป็นแพ็คเกจหลายครั้ง (เช่น "นวด Thai 10 ครั้ง — £450")

### ขายวอเชอร์
**Admin → 🎁 Vouchers → Sell**

**ประเภทวอเชอร์:**
- **💷 Money (วอเชอร์เงิน)** — ของขวัญแบบทั่วไป ลูกค้าใช้ยอดเงินซื้อ Treatment ไหนก็ได้ ตั้งวันหมดอายุได้
- **🎟 Sessions (วอเชอร์จำนวนครั้ง)** — แพ็คเกจ Treatment กำหนด Treatment (หรือ "Any treatment") + จำนวนครั้ง ลูกค้าจ่ายครั้งเดียว ใช้ครั้งละ 1 sessions ต่อการมาแต่ละครั้ง

**ฟิลด์ที่ต้องกรอก:**
- มูลค่าวอเชอร์ (ราคาขาย)
- สำหรับ sessions: จำนวนครั้ง
- ชื่อผู้ซื้อ (Purchased by)
- ชื่อผู้รับ (Gift for)
- **อีเมลผู้รับ** (ไม่บังคับ) — ถ้ากรอก ระบบส่งวอเชอร์ทางอีเมลอัตโนมัติด้วย template สวยๆ
- **วันหมดอายุ** (ไม่บังคับ)
- **ลูกค้าจ่ายอย่างไร?** — Cash / Card / Split (บังคับ)

หลังบันทึก รหัสวอเชอร์ (`SPA-XXXXXXXX`) จะปรากฏ ส่งให้ลูกค้าหรือแจ้งว่าส่งอีเมลแล้ว

### ใช้วอเชอร์ตอน Checkout
- แตะปุ่ม **🎁 Voucher**
- ใส่รหัส → Check
- กดยืนยัน Redeem

**วอเชอร์เงิน**: หักออกจากบิล min(ยอดวอเชอร์, ยอดบิล) ถ้าไม่พอ ต้องเก็บส่วนต่างผ่าน Cash/Card (ใช้ Split)

**วอเชอร์ sessions**: หัก 1 ครั้ง วอเชอร์ต้องใช้กับ Treatment ที่ถูกต้อง (หรือ "Any treatment") 1 วอเชอร์ = 1 บิล = 1 ครั้ง

### วอเชอร์ใน Split payment
สามารถใช้วอเชอร์เป็นแถวหนึ่งในการชำระ split ได้ (เช่น £40 วอเชอร์ + £25 บัตร) เลือก **🎁 Voucher** ในแถวหนึ่ง ใส่รหัส แถวจะขยายแสดงยอดคงเหลือ ระบบจะจำกัดจำนวนไม่ให้เกินวอเชอร์

## 12. แคมเปญอีเมล

ส่งอีเมลโฆษณาแบบมีแบรนด์ให้กับกลุ่มลูกค้าในระบบ

### กลุ่มลูกค้า (Segments)
- **VIP** — 10+ ครั้ง
- **Regular** — 3–9 ครั้ง
- **Lapsed** — ไม่มาเกิน 60 วัน
- **Treatwell** — ลูกค้าที่มาจาก Treatwell
- **All marketing-consented** — ทุกคนที่ยินยอมรับอีเมล

ส่งเฉพาะลูกค้าที่ **marketing_consent = TRUE** และยังไม่ unsubscribe ตามกฎหมาย GDPR

### ส่งแคมเปญ
**Admin → 📧 Campaigns**

1. เลือกกลุ่ม (พรีวิวจะแสดงจำนวนผู้รับ)
2. หัวข้อ + เนื้อหา HTML (ใช้ `{{name}}` เพื่อใส่ชื่อลูกค้า)
3. **Preview** เพื่อดูตัวอย่างก่อนส่ง
4. **Send** — Brevo จะส่งอีเมลทีละคน

ทุกอีเมลมีลิงก์ unsubscribe — กฎหมาย UK บังคับ คลิกแล้วจะบันทึก `unsubscribed_at` ถาวร (แม้ marketing_consent จะถูกเปิดอีกครั้ง การ unsubscribe ก็ยังชนะ)

### ประวัติแคมเปญ
หน้าเดียวกันจะแสดงแคมเปญที่ส่งไปแล้ว พร้อมวันที่ กลุ่ม และจำนวนผู้รับ

- **Booking source** — Treatwell vs widget vs walk-in
- **Therapist performance** — ตารางพนักงาน พร้อมคอลัมน์:
  - Bills (จำนวนบิล)
  - **Hours worked** (ชั่วโมงที่ทำงาน — รวมเวลาทุก Treatment)
  - **Requested hours** (ชั่วโมงที่ลูกค้าระบุชื่อขอเฉพาะ — สีทอง = ตัวชี้วัดความนิยม)
  - **%** — สัดส่วน Requested ต่อ total
  - Revenue · Tips · Total
- **📥 Export CSV** + **🖨 Print** ด้านบนขวา

### CSV exports ทั่วระบบ

## 15. แก้ปัญหาเบื้องต้น

### "เลื่อนหน้าจอไม่ได้บนมือถือ"
Hard refresh: **⌘ + Shift + R** บน Mac, **Ctrl + Shift + R** บน Windows ระบบใช้ viewport ที่รองรับมือถือ แต่ browser บางครั้ง cache layout เก่า

คุณกำลังดูเวอร์ชั่นเก่าที่ cache อยู่ Hard refresh (ดูข้างบน) Split modal ปัจจุบันรองรับ **Cash / Card / 🎁 Voucher** เป็น 3 ทางเลือกของแต่ละแถว

### "Override hours ไม่ผลในตารางเวลา"
Hard refresh แล้วดูหัวคอลัมน์ — ถ้าเห็น **★ ทอง** หน้าเวลา override แสดงว่าทำงานถูก ถ้าเห็นแต่ตัวเลขขาว (ตารางประจำสัปดาห์) แสดงว่า override ไม่ได้ถูกจับ — ติดต่อ support

### "ลูกค้าจ่ายออนไลน์แล้ว แต่ Checkout ยังถามราคาเต็ม"
เปิดการจองจาก timeline ตรวจดูแถบเขียว **"💳 Deposit £25 paid online · balance due at till"** ถ้าไม่เห็น แสดงว่ามัดจำไม่ได้ถูกบันทึก — เช็คที่ **Admin → 🌐 Online Booking**

### "Widget บนเว็บไซต์แสดง 'Stripe not configured'"
ตั้งค่า env vars บน Railway service:
- `STRIPE_PUBLISHABLE_KEY` — pk_live_… หรือ pk_test_…
- `STRIPE_SECRET_KEY` — sk_live_… / sk_test_…
- `STRIPE_WEBHOOK_SECRET` — จาก Stripe dashboard webhook
- `BOOKING_SECRET` — สร้างด้วย `openssl rand -hex 32`

แล้วใน Stripe → Webhooks ชี้ `https://spa-api.siamepos.co.uk/api/stripe/webhook` ไปยัง event `payment_intent.succeeded`

ถ้าไม่ตั้ง ระบบยังจองได้แต่ไม่เก็บมัดจำ (fallback)

### "ลูกค้าบอกว่าไม่ได้รับอีเมลยืนยัน"
- เช็คว่า `BREVO_API_KEY` ตั้งไว้บน Railway
- เช็คอีเมลลูกค้าใน **Admin → 👤 Clients** — บางครั้งพิมพ์ผิด
- ส่งซ้ำโดยแก้ไขการจองแล้วบันทึกอีก (ส่งใหม่อัตโนมัติ) หรือสร้างลิงก์ชำระเงินใหม่ถ้าเป็นการจองทางโทรศัพท์

### "อยากย้อนวันที่จองในอดีต"
ฟอร์มจองรับวันที่ใดๆ ก็ได้ สำหรับข้อมูลย้อนหลัง ใช้ปุ่ม +New แทน timeline (timeline แสดงวันนี้เป็นค่าเริ่มต้น — เปลี่ยน date picker ก่อน)

### "GDPR — ลูกค้าขอให้ลบข้อมูล"
**Admin → 👤 Clients → แตะแถว → แตะชื่อลูกค้า 5 ครั้ง** → ปุ่ม 🗑 จะปรากฏ → กดยืนยัน **ลบถาวร** ข้อมูลทั้งหมดของลูกค้า (การจอง ข้อมูลทางการแพทย์ ติดต่อ) ถูกลบจากฐานข้อมูล บิลเก่าจะยังคงรายการการเงินไว้แต่ถูกแปลงเป็นไม่ระบุชื่อ

### "Z-Report แสดงชื่อสปาผิด"
ตั้งใน **Admin → 🔧 Settings → Spa name** (ค่าใน settings ชนะ env var) Hard refresh
